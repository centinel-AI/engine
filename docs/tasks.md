# Task reference

All tasks are defined in [`Taskfile.yml`](../Taskfile.yml). Run `task --list` for a live summary.

---

## Setup

| Task | Description |
|------|-------------|
| `env:init` | Create `.env` from `.env.example`. No-op if `.env` already exists. |

---

## Codegen

Generates `providers/<cloud>/*.tf` from the pinned provider schema. Results are cached under `tmp/codegen/<cloud>/` and required by `generate:fixtures:*`.

| Task | Description |
|------|-------------|
| `generate:azure` | Regenerate `providers/azure/` from AzureRM + AzureAD schemas. |
| `generate:aws` | Regenerate `providers/aws/` from the AWS provider schema. |
| `generate:gcp` | Regenerate `providers/gcp/` from the Google provider schema. |
| `generate:oci` | Regenerate `providers/oci/` from the OCI provider schema. |
| `generate:all` | All four clouds in parallel. |
| `generate:fixtures:<cloud>` | Generate minimal test fixtures into `data/<cloud>/_test-all/`. Requires `generate:<cloud>`. |
| `generate:fixtures:all` | Fixtures for all clouds. |

---

## Build

Build engine Docker images. `generate:<cloud>` runs automatically as a dependency.

| Task | Description |
|------|-------------|
| `build:<cloud>:terraform` | Build the Terraform image for the given cloud. |
| `build:<cloud>:opentofu` | Build the OpenTofu image for the given cloud. |
| `build:all` | Generate all providers, then build all eight images in parallel. |

## Workspace

Populate `./workspace/` with the selected cloud module â€” the same layout baked into Docker images. Lets you run Terraform or OpenTofu directly on the host without Docker.

```bash
task workspace:azure:terraform PROJECT=project-01
terraform -chdir=workspace init
terraform -chdir=workspace plan
```

| Task | Description |
|------|-------------|
| `workspace:<cloud>` | Shorthand â€” populates with Terraform engine. |
| `workspace:<cloud>:terraform` | Populate `workspace/` with Terraform as the engine binary. |
| `workspace:<cloud>:opentofu` | Populate `workspace/` with OpenTofu as the engine binary. |
| `workspace:<cloud>:data` | Sync `data/<cloud>/<PROJECT>/` â†’ `workspace/data/` only, without re-populating providers. |

`PROJECT=<name>` overrides `ENGINE_TF_VAR_project` from `.env`.

## Run

Execute a built engine image via Docker Compose. Default command is `terraform plan` / `tofu plan`. Append `-- <command>` to run anything else.

```bash
task run:aws                              # plan with the project from .env
task run:aws PROJECT=networking           # override project
task run:aws -- terraform apply
task run:gcp:opentofu -- tofu destroy -auto-approve
```

| Task | Description |
|------|-------------|
| `run:<cloud>` | Shorthand â€” runs the Terraform container. |
| `run:<cloud>:terraform` | Run `engine-<cloud>-terraform`. |
| `run:<cloud>:opentofu` | Run `engine-<cloud>-opentofu`. |

`PROJECT=<name>` overrides `ENGINE_TF_VAR_project` for the container run.

## Tests

Terratest plan tests against `data/<cloud>/_test-all/` â€” fixtures generated by `generate:fixtures:<cloud>`. Each fixture covers one resource type. Tests skip automatically when the required credentials are absent.

| Task | Credentials required |
|------|----------------------|
| `test:plan:azure` | `ENGINE_ARM_CLIENT_ID`, `ENGINE_ARM_CLIENT_SECRET`, `ENGINE_ARM_TENANT_ID`, `ENGINE_ARM_SUBSCRIPTION_ID` |
| `test:plan:aws` | `ENGINE_AWS_ACCESS_KEY_ID`, `ENGINE_AWS_SECRET_ACCESS_KEY`, `ENGINE_AWS_DEFAULT_REGION` |
| `test:plan:gcp` | `ENGINE_GOOGLE_PROJECT`, `ENGINE_GOOGLE_CREDENTIALS` |
| `test:plan:oci` | `ENGINE_OCI_TENANCY_OCID`, `ENGINE_OCI_USER_OCID`, `ENGINE_OCI_FINGERPRINT`, `ENGINE_OCI_PRIVATE_KEY_FILE` |
| `test:plan:all` | All of the above. |
